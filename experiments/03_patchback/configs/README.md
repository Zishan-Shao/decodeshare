# Patchback Configs

Place patchback configs here. Configs should record:

- model and model dtype;
- layer and seed;
- basis tasks and target task;
- subspace rank and Q path;
- flip cap and evaluation size;
- patching direction and control set;
- output JSON/summary paths.

Suggested filename:

```text
model=<model_slug>__layer=<layer>__task=<task>__seed=<seed>.yaml
```
